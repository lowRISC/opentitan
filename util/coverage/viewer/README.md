# OpenTitan Coverage Viewer

The **OpenTitan Coverage Viewer** is a specialized web-based application
designed for high-assurance analysis of code coverage data. It provides an
interactive interface for exploring coverage results with features beyond
standard LCOV reports.

## Overview

The viewer is the primary tool for analyzing test coverage quality in the
OpenTitan project. It integrates data from C/C++, RISC-V Assembly, and OTBN code
into a unified interface.

### Key Features

-   **Per-Test Traceability:** Users can see exactly which test (or set of
    tests) exercised a specific line of code.
-   **Coverage View Filtering:** Automatically filters out code discarded by the
    linker to provide an accurate representation of the relevant firmware
    coverage.
-   **Target Filtering:** Isolate coverage for specific targets (e.g.,
    `ROM_EXT`) while hiding extraneous test framework code.
-   **Single-File Bundling:** Can be distributed as a standalone HTML file
    containing all necessary data and assets.

## Coverage View

Firmware analysis is often complicated by functions that are included in the
build but later stripped by the linker. The viewer uses the **Coverage View**
mechanism to address this.

### Coverage View Mechanism

A Coverage View is a baseline LCOV report created for a specific binary using
the `generate_coverage_view` tool. It combines two sources of truth:

1.  **LLVM Coverage Mapping:** Extracted from the ELF binary, providing metadata
    about function source regions.
2.  **DWARF Debug Information:** Generated by the compiler, identifying source
    lines that resulted in machine instructions in the final binary.

The tool considers a function **retained** if any of its source lines correspond
to an actual instruction in the DWARF info. Linker-discarded functions are
excluded from the view.

### Bazel Integration

To automate this process, use the `coverage_view_test` Bazel rule (defined in
`//rules/coverage:view.bzl`):

```bazel
load("//rules/coverage:view.bzl", "coverage_view_test")

coverage_view_test(
    name = "rom_ext_coverage_view",
    elf = ":my_rom_ext_binary",
    kind = "ibex", # or "otbn"
)
```

The name of the target **MUST** end with `_coverage_view`.

## Coverage Data

### Coverage Data Source

The viewer supports three methods for loading coverage data:

1.  **URL Parameter:** Fetch data from a `?cov=URL` parameter.
2.  **Embedded Data:** Load base64-encoded, gzipped data from the `bundledData`
    variable within the HTML (populated by `bundler.py`).
3.  **Default File:** Fetch `coverage.json.gz` from the same directory as the
    HTML file.

### Data Schema

The viewer consumes a compressed JSON format that includes per-test execution
metadata and source mapping. For technical details, see the
**[Data Schema Documentation](./SCHEMA.md)**.

## Generation Tools

### 1. Data Collection (`collect.py`)

Aggregates individual LCOV (`.dat`) files and coverage views into a single,
unified JSON file.

```bash
# Basic usage
python3 -m util.coverage.viewer.collect

# Advanced usage with overrides
python3 -m util.coverage.viewer.collect \
  --lcov_files=path/to/lcov_files.tmp \
  --source_dir=. \
  --output=path/to/coverage.json.gz
```

### 2. Report Bundling (`bundler.py`)

Embeds the compressed JSON data into the viewer HTML template to create a
standalone report.

```bash
# Basic usage
python3 -m util.coverage.viewer.bundler

# Advanced usage
python3 -m util.coverage.viewer.bundler \
  --coverage_json=path/to/coverage.json.gz \
  --output=path/to/index.html
```
