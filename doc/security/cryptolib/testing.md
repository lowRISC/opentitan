# OpenTitan Cryptography Library Testing

The cryptography library is subject to a variety of tests, including unit tests, functional tests (functests), and Known Answer Tests (KATs) with `cryptotest`.
This document provides an overview of the available tests, their locations, and how to run them.

The main directory for the cryptolib is `//sw/device/lib/crypto`. Tests are primarily located in `//sw/device/tests/crypto` and `//sw/otbn/crypto/tests`.

## Test Suites Overview

The cryptolib tests are organized into several Bazel test suites, which can be run with `bazelisk.sh test <suite_name>`. Device-specific tests require build and test tags to target a specific hardware platform (e.g., `--build_tag_filters=fpga_cw340_sival_rom_ext` and `--test_tag_filters=fpga_cw340_sival_rom_ext`).

### Host-Based Tests

These tests run on the host machine and do not require a hardware target.

-   **Implementation Unit Tests** (`//sw/device/lib/crypto/impl:impl_unittest_suite`): These are unit tests for the core cryptographic library primitives located in `//sw/device/lib/crypto/impl`.
-   **Base Library Unit Tests** (`//sw/device/lib/base:base_unittest_suite`): These are unit tests for the base functions that the cryptolib depends on.
-   **OTBN Simulation Tests** (`//sw/otbn/crypto/tests:cryptolib_sim_test_suite`): These tests run cryptographic operations for the OpenTitan Big Number Accelerator (OTBN) in a simulator.

### Device-Based Tests

These tests must be run on a specific hardware target, such as an FPGA or a silicon device.

-   **Driver Tests** (`//sw/device/lib/crypto/drivers:drivers_test_suite`): These are tests for the hardware drivers that interface with the cryptographic accelerators.
-   **Cryptolib Functional Tests** (`//sw/device/tests/crypto:cryptolib_test_suite`): These are functional, end-to-end tests for the cryptolib.
-   **RSA Key Generation Tests** (`//sw/device/tests/crypto:cryptolib_keygen_test_suite`): These tests specifically target the functionality of RSA key generation which are slow tests.
-   **Known Answer Tests (KATs)** (`//sw/device/tests/crypto/cryptotest:crypto_kat_test_suite`): These tests run a suite of Known Answer Tests (KATs) against the cryptolib primitives to ensure correctness. The skip-stride option can be used to test faster, e.g., `--test_arg=--skip-stride=40`.
-   **Base Library Functional Tests** (`//sw/device/lib/base:base_functest_suite`): These are functional tests for the base libraries.

## Coverage Generation

Code coverage can be generated by running the test suites with `bazel coverage` instead of `bazel test`. The `--config=ot_coverage` flag must be used to enable the correct compiler options.

To get a complete coverage picture, it is necessary to run three main groups of tests:
1.  **Host Tests**: The unit test suites (`impl_unittest_suite`, `base_unittest_suite`) and the OTBN simulation suite (`cryptolib_sim_test_suite`).
2.  **Device Tests**: The functional test suites, driver tests, and KATs.
3.  **OTBN Tests**: The OTBN crypto library coverage view.

The `otcrypto_coverage_view` target needs to be included as well. After running each group, the `ci/scripts/collect-coverage-report.sh` script should be used to gather the raw coverage data. Finally, `ci/scripts/merge-coverage-report.sh` can merge the data from all groups and generate a browsable HTML report. This information with commands can be found as comments above the `otcrypto_coverage_view` target.
